Form Hierarchy

Review of what might be done better

Challenge as I see it

The various Datasets we are given by customers are not under our control, tend to be quite large, sometimes deep in terms of information structure and can be quite onerous to fill in.

So

As a result a robust visual hierarchy is required which is flexible enough to cater for a deep structure (context is important) which can grow with a large input of data (eg repeating concepts)

Say that again

A solid visual hierarchy

is required to allow the User to easily understand the logical connections between the multiple fields on the form. As a result he/she can navigate the various data fields needing to be filled in with a low cognitive load. This should help to

  • eliminate mistakes
  • allow them to prioritise as they see fit

  • Its good that each field states a context "airway clearance" but this repetition comes at it own cost (the form is much wordier)
