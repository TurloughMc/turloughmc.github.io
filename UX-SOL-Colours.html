Colours

The aim of this presentation is to clarify our strategy in terms of colour usage within the interface.

See Desktop / Inspiration

Summary of Options

  • Separate - Background + Header + Side Menu
  • Grey + Black + White = Good + use Card for Content.
  • Grey + Black + Black = Good + use Card for Content
  • White + Black + Black = A little harsh and less good for mobile

Colour what?

  1. What you can click - links and table rows
  2. What we want you to click
  3. What you have clicked
  4. Where you are

Approach

  1. Links = BLUE and table rows = FONT COLOUR
  2. PRIMARY BUTTON or ACCENT COLOUR (preferred for variation)
  3. What you have clicked = PRIMARY COLOUR
  4. Where you are = FONT COLOUR
  5. unless its exceptually important = ACCENT

Accessibility & Colour

  1. Material colours are too heavily linked to Google branding, we shouldn't use them
  2. The colour chart linked in resources below are probably agnostic enough
  3. We should create a set of rules to limit useage of whatever colour set we use to prevent some Accessibility issues from the off - identify a palette set for which is contrast friendly
  4. Material design use a white on Primary and black on Secondary pairing, for now I'm suggesting we always go with white and dark
  5. Create a set of pairings for Notification Icons - measures off Red v Orange and Up Green V Down Red
  6. Colours need a back-up of signals for the colour blind - traffic lights most obviously - circle, triangle, octadecimal
