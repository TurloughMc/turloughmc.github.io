  3. User linkÂ - identifying the User and providing a link to additional links to do with that User
  4. Function menu - this provides options for the main branches of functionality provided by the System - Data Entry, Study Management, Service Desk

Domain

The Domain avalilable to a logged in User might be the entire System, a single Country, a Centre or in the case of Self Enrolment may be a the Patient Data under the control Patient Carer.

The Domain is different from the Context in that there may be multiple Contexts within a single Domain (thinks Centres in a Country or Patients within a Centre, Patients within a single Carers care). This means that the initial Context might match the Domain (a Carers sees an overview of all their Patients) but can Navigate down to a single Patient to focus on just one.

User link

  • Confirms who is logged in
  • Allows user to reset password
  • Define settings for that user

Function menu

Assuming that a particular User has the ability to do a number of different types of work - Data Entry, Data Management Administration etc. then they need a menu to switch between these

Context

Where the Domain scopes the parameters of what a User can explore within the System, the Context is what the User is actually looking at. When the User logs in the Context will be the top level of the Domain itself, but as he/she navigates down to more specific levels - Country > Centre > Patient, the conext changes

Context Banner

Is an interface object which allows the User to be sure of the Context the are looking at when the Navigate around the Registry. So for example this will display a Patients Name or ID

To be intuitive this needs to act consistantly with a heuristic model of the real world.

WTF?

The Context might be a Hotel, a Floor, a corridor, a room, a wardrobe. You can explore within the context more specifically (a wardrobe within a room). But to explore upwards (different room) or more generally (wardrobe in a different room) requires you to change context.

Context Menu

This provides a Menu to different areas of the Data for a given Context. For a Patient give might provide links to a Baseline form, a list of Encounters, End Study etc.
